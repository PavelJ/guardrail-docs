# GuardRail for Jira — Support & SLA

**Last updated:** April 19, 2026

## Support Channel

- **Email**: pavel.jira@gmail.com
- **Issue tracker**: [GitHub Issues](https://github.com/PavelJ/guardrail-docs-/issues)

## Response Times

GuardRail is provided on a best-effort support basis by an independent developer.

| Severity | Target initial response |
|----------|-------------------------|
| Critical (app unusable, data loss risk) | 1 business day |
| High (major feature broken) | 2 business days |
| Normal (bugs, questions, configuration help) | 3-5 business days |
| Low (feature requests, minor issues) | Best effort |

Business days are Monday–Friday, excluding Czech public holidays (CET / CEST timezone).

## Scope of Support

Support covers:

- Bug reports and defect resolution
- Configuration and usage questions
- Compatibility issues with Jira Cloud
- Security-related reports (please mark as "Security" in subject)

Support does not cover:

- Custom rule development on the user's behalf
- Integration with third-party systems not documented in the app
- Training or consulting services

## Supported Versions

Only the latest released version of GuardRail for Jira is supported. Users are automatically kept up to date via Atlassian Forge's managed deployment.

## Availability

GuardRail runs entirely on Atlassian Forge infrastructure. Application availability follows the [Atlassian Cloud SLA](https://www.atlassian.com/trust/compliance/sla). The developer does not operate independent servers or databases for GuardRail; all data is stored in Atlassian-managed storage.

## Security Issues

Report suspected security vulnerabilities directly to **pavel.jira@gmail.com** with the subject line "Security — GuardRail". Please do not open public issues for security matters. Critical security reports are prioritised above normal support requests.
